## 왜 GitHub 블로그인가
티스토리는 웹에서 쓰고 발행하는 흐름이 익숙해지기 전엔 생각보다 번거로웠어요. 그래서 **로컬에서 글 쓰고 git push로 발행**하는 방식으로 바꿉니다.

## 이 블로그에서 다룰 것
- **배운 것**: 개발/운영 중 새로 알게 된 것
- **만든 것**: 작은 자동화, 도구, 스크립트
- **회고(실패노트)**: 삽질/실수/개선 포인트

## 글 쓰는 규칙
- 결론 → 맥락 → 체크리스트
- 개인정보/민감정보는 원칙/패턴으로 익명화
- 짧게 자주, 가끔 깊게

첫 연재는 “세션 로그를 블로그 글로 바꾸는 루틴”부터 정리해볼게요.
